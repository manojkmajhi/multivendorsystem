## 🎯 Key Points

### Magic Link vs OTP (Why we're switching):

| Issue | OTP Problem | Magic Link Solution |
|-------|------------|-------------------|
| **Rate limit** | 429 error after 3 requests | High limits, no throttling |
| **Empty emails** | Template variables broken | HTML template works perfectly |
| **User experience** | Copy 6-digit code | Click email link |
| **Security** | OTP can be intercepted | Single-use token |

### Files You'll Be Editing:

1. **server.js** - Add 5 new/updated endpoints
2. **NOTHING ELSE!** - Frontend is ready, docs complete

## 💡 How It Works (Quick Reference)

```
User Flow:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User visits /farmers/signup
   ↓
2. Enters email: test@cropsay.com
   ↓
3. Clicks "Send Verification Link"
   ↓
4. POST /api/farmers/send-magic-link
   → Supabase generates token
   → Email sent with magic link
   ↓
5. User receives email:
   Subject: "Verify with the otp"
   Contains: Magic link button
   ↓
6. User clicks link
   ↓
7. GET /farmers/signup-verify?token_hash=xxx
   → Token verified
   → Temp cookies set
   → Profile form shown (farmer-signup-complete.ejs)
   ↓
8. User fills profile (name, phone, password)
   ↓
9. Clicks "Complete Signup"
   ↓
10. POST /api/farmers/complete-signup
    → Check cookies
    → Create farmer in database
    → Create session
    → Redirect to dashboard
    ↓
11. ✅ USER LOGGED IN & ON DASHBOARD
```
